# 进程和线程的区别

> 进程是操作系统调度的基本单位，线程是 CPU 调度的基本单位。

**进程**之间相互独立、拥有自己独立的内存空间和系统资源，切换开销较大，一个进程崩溃不会影响其他进程；**线程**会共享进程中的内存和资源，可以在多核 CPU 上并发执行，切换开销小，线程崩溃会导致进程崩溃；

```txt
+----------------- -+        +-------------------+
|      进程 A       |        |      进程 B       |
| +---------------+ |        | +---------------+ |
| |   线程 A1     | |        | |   线程 B1     | |
| +---------------+ |        | +---------------+ |
| |   线程 A2     | |        | |   线程 B2     | |
| +---------------+ |        | +---------------+ |
| |   线程 A3     | |        | |   线程 B3     | |
| +---------------+ |        | +---------------+ |
+-------------------+        +-------------------+
```
